# 产品概述

## 项目定位

个人使用的 Apple Health 数据处理管道,将原始 Apple Health 导出数据转换为结构化、可查询的数据集。

## 核心功能

### 数据摄入 (Ingestion)
- 解析 Apple Health `export.xml`
- 解析运动路线 GPX 文件
- 数据验证和清洗

### 数据转换 (Transformation)
- 标准化数据格式
- 提取关键指标
- 计算派生字段

### 数据存储 (Storage)
- 结构化数据持久化
- 索引优化
- 数据版本管理

### 数据导出 (Export)
- 多种格式导出 (JSON, CSV, Parquet)
- 查询接口
- 数据可视化支持

## 用户场景

1. **定期同步**: 从 iPhone 导出最新健康数据,更新本地数据库
2. **趋势分析**: 查询长期健康指标变化趋势
3. **运动分析**: 分析运动路线、配速、心率等数据
4. **数据导出**: 导出特定时间段的数据用于深度分析

## 非功能性需求

- **性能**: 处理 5 年历史数据 (约 100MB XML) 在 2 分钟内完成
- **可靠性**: 数据零丢失,每次处理生成验证报告
- **可维护性**: 清晰的数据模型,易于扩展新的数据类型
- **可观测性**: 详细的处理日志和指标

## 约束条件

- 个人使用,无需多用户权限管理
- 本地部署,数据不离开用户机器
- 支持 macOS 和 Linux
- Python ≥ 3.10
