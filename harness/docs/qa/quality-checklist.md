# 质量检查清单

## 代码提交前检查

### 必做项 (MUST)

- [ ] **Lint 通过**: `bash scripts/lint-all.sh` 全部通过
- [ ] **测试通过**: `pytest tests/ -v` 无失败
- [ ] **类型检查**: `mypy src/ --strict` 无错误
- [ ] **文档更新**: 相关文档已同步更新
- [ ] **不变性**: 所有架构不变性未被破坏

### 推荐项 (SHOULD)

- [ ] **测试覆盖**: 核心逻辑覆盖率 ≥ 80%
- [ ] **性能测试**: 大数据集测试无明显退化
- [ ] **代码审查**: 已请求审查或自我审查
- [ ] **Commit 规范**: Commit 消息符合约定

### 可选项 (MAY)

- [ ] **基准测试**: 新功能有性能基准
- [ ] **文档完善**: 使用示例和教程已更新

- [ ] **原始数据**: `raw/` 目录未被修改

## 技术债务清理

### 识别

- [ ] **代码异味**: 复杂度过高、重复代码
- [ ] **TODO 标记**: 未完成的实现
- [ ] **已知问题**: `known-issues.md` 中的问题

### 优先级

**高优先级** (立即处理):
- 安全漏洞
- 数据损坏风险
- 严重性能问题

**中优先级** (本迭代):
- 不变性违反
- 测试覆盖不足
- 错误处理缺失

**低优先级** (技术债务周):
- 代码风格不一致
- 文档不完善
- 小重构机会
